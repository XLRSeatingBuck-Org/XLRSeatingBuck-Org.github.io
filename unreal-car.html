Immersive Car Simulation in Unreal Engine

Experience driving through the diverse landscapes of farmlands, woods and suburban areas in Unreal Engine.

Key Features

This project aims to implement a framework for designing a suburban area by integrating Geographic Information System (GIS) datasets using procedural modeling approach . The chosen model area is in Corvallis, Oregon (see map), selected for its diverse landscapes â€”including natural areas (grasslands, woodlands, scrub), land uses (farmland, farmyards), and suburban housing. The ultimate goal is to support our immersive driving simulator that reconstructs realistic environments procedurally.

Immersive Driving Experience

Navigate, explore, or drive through a procedurally generated environmentâ€”customized to your preferences. Whether you're using a VR headset or a traditional monitor, and whether you want passive exploration or active simulation, the system is built to adapt to your setup and interests.

Real-World Reconstruction

The simulation environment is reconstructed from real-world geographic data sourced from OpenStreetMap. Using GIS attributes such as road classifications, land use types, and building footprints, we generate accurate 3D models that reflect the layout and structure of Corvallis. This data-driven approach ensures that the environment is both believable and scalable, enabling rapid generation of suburban areas without compromising on realism or detail.

Procedural Content Generation

From suburban houses, powerlines to a vast areas of farmlands, woods, the environment is built using rule-based procedural modeling, allowing streets, houses, fences, and infrastructure to be automatically generated from attributes and variables. This approach minimizes manual design work, speeds up iteration, and ensures consistent, scalable worldbuilding across large areas

Technical Workflow

The project integrates two primary data layers: (1) Geospatial Data from OpenStreetMap (OSM), which includes 2D boundary and attribute data; (2) 3D Procedural Content, generated using a combination of CityEngine and Unreal Engineâ€™s PCG (Procedural Content Generation) plugin.

1. Data Extraction & Modeling in CityEngine

CityEngine serves as the main tool for procedural generation of city elements such as streets, buildings, and decorations. It uses a robust library of CGA rule packages to automate 3D asset generation.

a) Extract 2D Data Layers from OpenStreetMap

  • Shapes: Represent buildings, land uses, and parking zones.
  • Graphs: Represent roads (highways, cycleways, footways) and linear infrastructure (fences, powerlines).
  • These elements include object attributes that drive procedural rules.

b) Apply Rule Packages to Generate 3D Content

Street Creation:

Uses the Complete Street Rule Package to generate streets and sidewalks based on number of lanes, street direction, cycleway, and parking attributes.

Suburban House Block:

Identify enclosed street networks that define blocks.
Convert blocks into shapes and subdivide into front-facing lots.
Apply the Suburban House Block rule to procedurally generate homes with consistent spacing and orientation.

Additional Elements:

Solar panels, fences, and other suburban features are added using additional rule packages.

2. From CityEngine to Unreal Engine

CityEngine models can be exported directly into Unreal Engine using the Datasmith plugin. This integraion retains all object attributes associates with each shapes/meshes. These object attributes are converted to tags serving as key inputs for PCG graph

Procedural Content Generation plugin

The PCG system in Unreal Engine provides a cabapilities to create your own procedural content inside the Unreal Engine. PCG logic is defined in PCG Graph. We create separate PCG Graphs for each landscapes : wood, farmland, grass, meadow, road. General key: all PCG graphs implement the World Ray Hit Query to scatter points all over the meshes with the specific tag (wood/farmland/grass/...) and project all the points onto the landscape data. At each points, we assign random offsets, scales, and rotations to give a more random, realistic looking scene, then we spawn different static meshes at those points.
